📖 HANA World Admin 사용 가이드

⬅ 대시보드로

1) 준비물

  • 브라우저 + MetaMask 또는 Trust Wallet
  • 네트워크:
    • Binance Smart Chain Mainnet (Chain ID: 56) - 실제 운영 네트워크
  • 컨트랙트 주소
  • 충분한 BNB (가스비용)
    • 메인넷: 실제 BNB 필요 (거래소에서 구매 가능)

2) 초기 설정

  1. smartcontract.js에 컨트랙트 주소를 입력합니다.
  2. index.html을 열고 상단에서 BSC 메인넷 선택 → 지갑 연결을 진행합니다.
  3. 우측 상단의 권한 레벨이 OWNER인지 확인합니다.
  4. 네트워크 자동 전환: 선택한 네트워크와 지갑이 다르면 자동으로 전환을 요청합니다.

3) 컨트랙트 상태

현재 토큰 상태를 한눈에 확인하는 섹션입니다. 상태 새로고침으로 최신 상태를 가져오고, 빠른 제어 버튼으로 pause, unpause를 즉시 전환할 수 있습니다.

  • name: HANA World, symbol: HWT, decimals: 18
  • totalSupply (총 발행량): 1,000,000,000 HWT
  • paused (일시정지 여부)
  • owner (컨트랙트 소유자)

4) Lockup 관리

⚠️ 매우 중요: 락업은 "덮어쓰기" 방식입니다!

setLockup()과 decreaseLockAmount()는 기존 값을 완전히 새로운 값으로 교체합니다!

  • 시나리오 1:
    • 현재 락업: 1000 HWT
    • setLockup(주소, 시간, 100)을 실행
    • 최종 결과: 100 HWT (❌ 1100 아님!)
  • 시나리오 2 (감소):
    • 현재 락업: 1000 HWT
    • decreaseLockAmount(주소, 700)을 실행
    • 최종 결과: 700 HWT (❌ 300 아님!)
  • 시나리오 3 (부분 해제 - 차감 방식):
    • 현재 락업: 1000 HWT
    • releaseLockup(주소, 300)을 실행
    • 최종 결과: 700 HWT (✅ 차감됨!)

💡 핵심 요약:

  • setLockup(): 완전히 새로운 락업으로 교체
  • decreaseLockAmount(): 새로운 수량으로 교체 (현재보다 작아야 함)
  • releaseLockup(): 입력한 수량만큼 차감 (유일하게 빼기 방식)
  • extendLockup(): 시간만 추가 (수량 불변, 만료 전에만 가능)

📋 각 락업 함수 상세 설명

  • 새 락업 설정 (setLockup): 특정 주소의 토큰을 일정 기간 동안 잠급니다
    • 시간: 분 단위로 입력 (정수만 가능)
    • 수량: 소수점 입력 가능 (예: 100.5 HWT)
    • 기존 락업이 있으면 완전히 새 값으로 교체됨!
  • 락업 연장 (extendLockup): 기존 락업 기간을 추가로 연장합니다 (분 단위 정수)
    • 수량은 변경되지 않고 시간만 추가됩니다
    • 락업이 없는 주소에는 사용 불가
    • 만료된 락업은 연장 불가 (LockupExpired 에러 발생)
  • 락업 수량 감소 (decreaseLockAmount): 락업된 수량을 줄입니다
    • 소수점 가능, 0으로 설정 시 전체 해제
    • 현재 수량보다 작은 값만 가능하며, 기존 값에서 빼는 것이 아니라 새 값으로 교체!
    • 예: 1000 → 700 입력 → 최종 700 (300이 아님!)
  • 락업 부분 해제 (releaseLockup): 락업 중 일부 수량만 해제합니다
    • 소수점 가능
    • 입력한 수량만큼 기존 락업에서 차감됩니다 (유일한 빼기 방식!)
    • 예: 1000 락업 → 300 해제 → 최종 700 락업
  • 만료된 락업 제거 (clearExpiredLockup): 기간이 만료된 락업 정보를 삭제합니다
    • 필수 작업이 아닙니다 - 만료되면 자동으로 효력이 없어집니다
    • 단지 컨트랙트 저장소를 깔끔하게 정리하는 용도
    • 만료되지 않은 락업에는 사용 불가 (에러 발생)

보안: Owner만 락업 관리 기능을 실행할 수 있습니다.

기간 분(Minutes) 용도
1시간 60 테스트용
1일 1,440 단기 락업
1주일 10,080 일주일 이벤트

💰 토큰 수량 예시 (소수점 가능)

  • 정수: 100 HWT, 1000 HWT, 1000000 HWT
  • 소수점 1자리: 100.5 HWT, 999.9 HWT
  • 소수점 2자리: 99.99 HWT, 1234.56 HWT
  • 소수점 많은 자리: 123.456789 HWT, 0.00001 HWT

5) Lockup 조회

  • 락업 정보 확인: 주소의 락업 수량, 남은 시간, 만료 시각 조회
  • 사용 가능 잔액 확인: 전체 잔액 중 락업되지 않은 수량 조회
  • 가스비가 들지 않는 조회 함수입니다

6) 토큰 전송

  • 자신의 토큰을 다른 주소로 전송합니다
  • 락업된 토큰은 전송할 수 없습니다
  • 전송자는 충분한 잔액과 BNB(가스비)를 보유해야 합니다

7) 토큰 소각

  • 소각: 자신의 토큰을 영구적으로 제거합니다
  • 소각된 토큰은 복구할 수 없으므로 신중하게 사용하세요
  • 총 공급량이 감소합니다

8) 토큰 정보 조회

  • 토큰 잔액 조회: 특정 주소의 HWT 보유량 확인
  • 가스비가 들지 않는 읽기 전용 함수입니다

오류 해결

  • "Cannot read properties of undefined (reading 'eth')"
    • 원인: 지갑이 연결되지 않은 상태에서 조회 기능 사용
    • 해결: 먼저 MetaMask 또는 Trust Wallet 지갑을 연결한 후 조회하세요
  • "현재 네트워크에서 컨트랙트를 찾을 수 없습니다"
    • 원인: 잘못된 네트워크 선택 또는 컨트랙트 주소 불일치
    • 해결: smartcontract.js의 주소와 BSC 메인넷(Chain ID: 56) 선택을 확인하세요
  • "가스 추정에 실패했습니다"
    • 원인: 입력값 오류, 권한 부족, 또는 컨트랙트 상태 문제
    • 해결: 입력값, 권한(Owner), BNB 잔액을 확인하세요
  • "락업이 이미 만료되었습니다 (LockupExpired)"
    • 원인: 만료된 락업을 연장하려고 시도
    • 해결: 만료된 락업은 연장할 수 없습니다. 새로운 락업을 설정하세요

보안 권장

  • Owner 권한은 최소한으로 관리하세요
  • 중요한 작업은 멀티시그/하드웨어 월렛 권장
  • 컨트랙트 주소를 항상 재확인하세요 (피싱 방지)
  • 락업 설정 시 시간과 수량을 반드시 재확인하세요
  • 대량의 토큰 작업 전 소량으로 먼저 테스트하세요
  • ⚠️ 메인넷에서 직접 작업하므로 모든 트랜잭션은 실제 BNB가 소모되고 복구 불가능합니다
  • 중요 작업 전 컨트랙트 함수를 "call"로 시뮬레이션하여 결과를 미리 확인하세요

  • 트랜잭션 검증: 모든 트랜잭션의 상태 및 가스 소모량 확인

추가 도움말

  • MetaMask에 BSC 네트워크 추가:
    • 네트워크 이름: Binance Smart Chain
    • RPC URL: https://bsc-dataseed.binance.org/
    • Chain ID: 56
    • 통화 기호: BNB
    • 블록 탐색기: https://bscscan.com
  • BscScan: 메인넷 탐색기에서 모든 트랜잭션 확인 가능
  • 가스비 절약: 네트워크가 붐비지 않은 시간대에 트랜잭션 전송 (보통 새벽 시간대)
  • BNB 구매: 바이낸스, 업비트, 빗썸 등 한국 거래소에서 구매 후 MetaMask로 전송
  • 문의: 컨트랙트 관련 문의는 개발팀에 문의하세요

⚠️ 메인넷 운영 시 주의사항

  • 테스트넷이 없으므로 모든 작업이 실제 블록체인에 기록되고 복구 불가능합니다
  • 소량의 토큰으로 먼저 테스트한 후 대량 작업을 진행하세요
  • 락업 설정 시 시간과 수량을 여러 번 확인하세요 (실수 시 복구 불가)
  • 컨트랙트 주소를 반드시 BscScan에서 확인하세요 (피싱 방지)
  • Owner 개인키는 하드웨어 월렛 또는 멀티시그로 보호하세요
  • 중요한 트랜잭션 전 가스비 여유분을 확보하세요 (최소 0.01 BNB 이상)
